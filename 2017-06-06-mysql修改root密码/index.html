mysql修改root密码

  • 2017-06-06
  • ·
  • 运维
  • ·
  • Views

一直没有从零开始倒腾过mysql，用的都是别人配置好的东西，安装、配置、修改root密码这些都没做过，就试了一下，坑比较多，需要填

操作系统：macOS 10.12.4

Mysql：MySQL Community Server 5.7.18

first：下载安装包，mysql官网下载

second：安装，首次安装到最后会提示root的初始密码，这个密码记不记不太重要

third：修改root密码

1. sudo mysqld_safe --skip-grant-tables，进入安全模式
2. mysql -u root -p ，输入任意密码登录root
3. UPDATE mysql.user SET authentication_string=PASSWORD(‘123456’) where User=’root’; //将root用户密码改成 123456
4. flush privileges;  刷新权限，使配置生效
5. 输入mysql -u root -p命令，接着输入刚刚设置的密码，如果可以登录说明修改成功，则不需要进行下一步
6. 如果在第5步中出现your password has expired. to log in you must change it using a client that supports expired passwords错误，说明密码过期需要将密码过期字段设置为‘N’
7. use mysql 
8. select * from mysql.user where user='root' \G
9. update user set password_expired='N' where user='root';
10. flush privileges;  刷新权限，使配置生效
最后退出mysql，重新mysql

折腾完mac上的mysql之后，又倒腾了一下centos server上的mysql

操作系统：CentOs 7.3

Mysql: 7.5.6

下载的是二进制包，直接解压缩即可

first：创建mysql用户账号，useradd -s /sbin/nologin -M mysql

second：修改配置文件，配置文件一般放在/etc下，/etc/my.cnf

third：初始化数据库

forth：修改密码，参考mac部分

fifth：配置远程连接

至此，完美结束！

如果不可以进行远程连接，可以尝试一下解决方案：

查看mysql置文件

vi /etc/my.cnf

特别要留意其中的两项：bind_address和skip_networking,bind_address一定不能为127.0.0.1，否则只能在本地连接，skip_networking一定不能出现，否则只接受unix socket而不能提供tcp socket服务，建议将bind_address和skip_networking直接都注释掉。
