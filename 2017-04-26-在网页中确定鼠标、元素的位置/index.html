在网页中确定鼠标、元素的位置

  • 2017-04-26
  • ·
  • JavaScript
  • ·
  • Views

在需要获取鼠标相对于元素的位置的时候，首先必须确定鼠标位置、元素位置，如此才能进行计算。鼠标位置、元素位置有多种获取方式，获取的值也各不相同，有的是相对于视口的，有的是相对于文档的….在没有滚动条的时候计算会很简单，相对于文档和相对于视口取得的值相同，一旦有了滚动条就会变得麻烦起来。

获取鼠标位置

  • pageX、pageY

    鼠标事件的只读属性(MouseEvent.pageX ，MouseEvent.pageY)pageX和pageY返回的是相对于整个文档的坐标，以像素为单位。

    This property takes into account any vertical(pageX) , horizontal(pageY)scrolling of the page. ——from MDN

    这两个属性会把滚动也计算进去。但是这两个属性只会把body的滚动计算进去，如果是body内部元素的滚动需要在pageX、pageY上叠加上滚动的值。

    其实这是一个有意思的事情。

  • clientX、clientY

    鼠标事件的只读属性clientX和clientY返回的是相对于浏览器内容区域的X、Y坐标，不包含滚动的值

    clicking in the top-left corner of the client area will always result in a mouse event with a clientX value of 0, regardless of whether the page is scrolled . ——from MDN

  • screenX、screenY

    鼠标事件的只读属性screenX和screenY返回的是鼠标相对于屏幕的距离。

  • offsetX、offsetY

    和pageX、pageY类似，只有IE支持

获取DOM元素位置和尺寸大小

以下内容都是DOM元素的属性

  • offsetWidth 和 offsetHeight：元素宽度，包括border和padding

  • clientWidth 和 clientHeight：元素宽度，包括padding，不包含滚动条，margin，border。如果没有CSS属性或者是inline layout box其值将会是0

  • scrollWidth和scrollHeight：元素的内容区域加上内边距加上溢出尺寸，当内容正好和内容区域匹配没有溢出时，这些属性与clientWidth和clientHeight相等

  • offsetLeft 和 offsetTop：和已定位父容器左上角的距离。

    注意：对于块级元素来讲，offsetWidth、offsetHeight、offsetLeft、offsetTop描述的是一个元素border的宽度高度以及相对于offsetParent的距离

    However, for inline-level elements (such as span) that can wrap from one line to the next, offsetTopand offsetLeft describe the positions of the first border box (use Element.getClientRects() to get its width and height), while offsetWidth and offsetHeight describe the dimensions of the boundingborder box (use Element.getBoundingClientRect() to get its position). Therefore, a box with the left, top, width and height of offsetLeft, offsetTop, offsetWidth and offsetHeight will not be a bounding box for a span with wrapped text.

  • clientTop和clientLeft返回内边距的边缘和边框的外边缘之间的水平和垂直距离，也就是左，上边框宽度

  • scrollLeft和scrollTop是指元素滚动条位置
